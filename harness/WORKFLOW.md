# Backend Agent Workflow

## 1. 계획 수립 및 승인 (Plan & Approval)
- `CLAUDE.md`의 '문서 지도'를 참고하여 현재 작업에 필요한 문서를 읽는다.
- 작업의 목표, 생성/수정할 파일 목록, 예상되는 사이드 이펙트를 포함하여 `PLAN.md`를 작성하고 사용자에게 보고한다.
- 만약 domain/ 영역의 수정이 포함된다면, PLAN.md의 상단에 [도메인 수정 승인 요청] 섹션을 별도로 작성하여 변경의 필연성을 논리적으로 설명하라.
- 명시적인 승인(Approve)이 떨어지기 전까지는 절대 코드를 작성하거나 수정하지 않는다.

## 2. 브랜치 생성 (Branching)
- 승인이 떨어지면 메인 작업 브랜치에서 분기하여 새로운 브랜치를 생성하고 이동한다.
- 브랜치 네이밍: `ai/feat-[기능명]` 또는 `ai/fix-[버그명]`

## 3. 구현 및 무한 루프 방지 (Implementation)
- 코드를 작성한다. 테스트 코드에 대한 내용은 `TESTING-PROTOCOL.md`를 참고한다
- [에러 통제]: 컴파일 에러나 예외 발생 시 스스로 원인을 분석하여 수정한다. 단, 동일한 에러로 3회 이상 해결에 실패할 경우, 코드 수정을 즉각 중단하고 사용자에게 상황을 보고한 뒤 지시를 대기한다.

## 4. 커밋 및 보고 (Commit)
- 작업이 완료되고 테스트가 통과하면, 변경 사항을 요약하여 커밋 메시지를 작성하고 스테이징 및 커밋한다.
