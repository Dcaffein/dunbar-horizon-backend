# Task Spec Guide

"무엇을, 어떻게" 구현할지는 에이전트가 코드베이스를 분석하여 PLAN.md에서 도출합니다.
이 명세서(Task Spec)에는 에이전트가 코드만 보고는 알 수 없는 맥락과 확정된 제약만 작성합니다.

## 섹션

| 섹션 | 필수 | 설명 |
|------|------|------|
| Background | — | 이 작업이 나오게 된 배경, 도메인 철학, 기획적 의도 |
| Objective | ✓ | 현재의 문제점과 이 작업을 통해 달성하려는 명확한 결과 |
| Decision | — | 인간이 사전에 확정한 절대 변경하면 안 되는 기술적 정책 |
| Open Question | — | 에이전트가 코드 분석 후 PLAN.md에서 장단점을 비교해 주었으면 하는 항목 |
| Resolution | — | (구현 완료 후 작성) 실제 수정된 핵심 파일과 구현 결과 요약 |
